# Security Policy

Linux101 — BootToRoot is an educational repository focused on Linux administration, infrastructure operations, and security-related technical learning.

This repository does not provide production support.

### Responsible Usage

All configurations, commands, labs, and infrastructure procedures provided in this repository are intended for:

- lab environments
- educational environments
- personal development environments

Always validate configurations before applying them to production systems.

### Sensitive Information Policy

Contributors must never upload:

- passwords
- API keys
- private keys
- internal IP inventories
- confidential infrastructure data
- production credentials
- customer information

Any accidental exposure should be reported immediately.

### Vulnerability Reporting

If you identify:

- insecure examples
- vulnerable configurations
- exposed credentials
- dangerous misconfigurations
- outdated security recommendations

please report them through repository issues or contact the maintainer.

### Security Scope Limitations

This repository may include:

- firewall examples
- authentication examples
- hardening examples
- networking configurations

These examples may require additional validation depending on:

- organizational policies
- compliance requirements
- production architecture
- infrastructure scale

### Educational Disclaimer

Security examples are provided for educational purposes.

Always perform independent validation before applying security configurations to production systems.
